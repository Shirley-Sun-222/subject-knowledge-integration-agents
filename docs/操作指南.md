# 系统操作指南

本文档面向评审、协作者和后续维护者，说明如何在本地、Docker 和公开部署环境中运行本项目，以及如何复现实验与 P2 技术报告结果。

## 1. 运行前提

推荐环境：

- Python `3.11+`
- Node.js `20+`
- npm
- macOS / Linux

可选但常用依赖：

- Tesseract：扫描版 PDF OCR
- Playwright Chromium：高保真 PDF 导出

项目的主要配置项见根目录 [`.env.example`](/Users/sunxueli/CodexFile/subject-knowledge-integration-agents/.env.example:1)。

与大文件解析和全量图谱直接相关的关键配置包括：

- `PDF_TEXT_EXTRACT_WORKERS`
- `PDF_OCR_WORKERS`
- `PARSE_CACHE_ENABLED`
- `GRAPH_MAX_CHAPTERS`
- `GRAPH_FULL_LLM_MIN_CHARS`
- `GRAPH_FULL_LLM_MIN_KEYWORDS`

## 2. 本地开发运行

### 2.1 安装依赖

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
npm install
npm --prefix frontend install
```

### 2.2 启动开发环境

```bash
npm run dev
```

默认地址：

- 前端：`http://localhost:5173`
- 后端健康检查：`http://localhost:8000/api/health`

如果只启动后端：

```bash
uvicorn backend.app.main:app --reload --host 0.0.0.0 --port 8000
```

## 3. Docker 运行

```bash
cp .env.example .env
docker compose up --build
```

本地 Docker 默认访问地址：

- `http://localhost:8000`

说明：

- 容器内部监听 `7860`，宿主机通过 `docker-compose.yml` 映射到 `8000`
- 运行时数据写入本地 `data/` 与 `report/`，不会写入镜像层

镜像变体：

- `light`：无 OCR、无浏览器 PDF，最小体积
- `standard`：有 OCR、无浏览器 PDF，当前公网主线
- `full`：有 OCR、浏览器 PDF 全量能力

## 4. 公网部署说明

当前公开部署主线是 ModelScope Docker 创空间。

已知公网入口：

- [ModelScope 部署](https://shirley222-subject-knowledge-integration-agents.ms.show)

部署脚本入口：

```bash
npm --prefix frontend install
npm --prefix frontend run build
bash scripts/start_modelscope.sh
```

部署注意事项：

- 公网部署必须使用容器化后端，`GitHub Pages` 不能承载完整系统
- `localhost` 不是比赛提交可接受的在线链接
- 若需要持久化运行态数据，应把 `DATABASE_URL`、`UPLOAD_DIR`、`INDEX_DIR`、`GENERATED_DIR` 指向持久化卷

## 5. 官方教材与本地实验

官方教材目前存放在本地 `textbooks/` 目录：

- `01_局部解剖学.pdf`
- `02_组织学与胚胎学.pdf`
- `03_生理学.pdf`
- `04_医学微生物学.pdf`
- `05_病理学.pdf`
- `06_传染病学.pdf`
- `07_病理生理学.pdf`

这些 PDF 仅用于本地实验和技术报告复现：

- 不提交到 GitHub
- 不同步到公开仓库
- 不写入对外文档的附件清单

现有导入脚本支持直接读取该目录：

```bash
python3 scripts/load_book_samples.py --sample-dir textbooks --reset-samples
```

当前 PDF 解析策略：

- 优先读取 PDF TOC/书签，只保留顶层教学章
- 没有可用 TOC 时，回退到正文正则切章
- `digital` PDF：只做原生文本抽取
- `mixed` PDF：只对稀疏页做并行 OCR
- `scanned` PDF：整本并行 OCR
- 若 `PARSE_CACHE_ENABLED=1`，相同文件哈希可跨工作区复用解析结果

## 6. Benchmark 与 P2 实验

### 6.1 通用 RAG benchmark

默认 benchmark：

```bash
python3 scripts/run_rag_benchmark.py
```

医学题集 benchmark：

```bash
python3 scripts/run_rag_benchmark.py \
  --question-set scripts/benchmark_sets/medical_official_questions.json
```

说明：

- 默认题集是旧的数据结构/算法问题
- 医学题集位于 `scripts/benchmark_sets/medical_official_questions.json`
- benchmark 输出默认写入 `data/generated/`

### 6.2 P2 官方教材实验

完整实验脚本：

```bash
python3 scripts/run_p2_rag_experiments.py --sample-dir textbooks --mode all
```

常用模式：

只跑官方子集：

```bash
python3 scripts/run_p2_rag_experiments.py --sample-dir textbooks --mode subset
```

只跑 7 本全量，沿用已选出的分块尺寸：

```bash
python3 scripts/run_p2_rag_experiments.py \
  --sample-dir textbooks \
  --mode full \
  --final-chunk-size 300
```

当前实验约定：

- 调试子集：`03_生理学`、`04_医学微生物学`、`05_病理学`
- 全量正式集：`textbooks/` 下全部 7 本教材
- 分块敏感性实验：`300 / 500 / 700 / 900`
- 当前保守实验环境：`hash embedding fallback + 无在线 LLM`

图谱模式约定：

- `预览图谱`：允许截断，适合大教材快速反馈
- `全量图谱`：不再受 `GRAPH_MAX_CHAPTERS` 限制，必须处理全部章节
- 全量图谱采用“全覆盖分层抽取”：
  - 高信息量章节优先走 LLM
  - 低信息量、目录噪声和边角章节走快速抽取

实验输出位置：

- `data/generated/p2-rag-official/subset/`
- `data/generated/p2-rag-official/full/`

这些目录包含：

- 消融 JSON
- Markdown 表格
- chunk size 对比结果
- manifest 汇总

## 7. P2 技术报告使用方式

P2 草稿位于：

- [P2技术报告.md](/Users/sunxueli/CodexFile/subject-knowledge-integration-agents/docs/P2技术报告.md:1)

## 8. 对外公开与隐私边界

允许公开：

- 源代码
- 聚合后的实验结论
- Markdown 技术报告草稿
- 公开部署链接

不允许公开：

- `textbooks/` 官方教材 PDF
- `data/` 下的原始运行产物
- `.env`、`.env.modelscope`
- 任何真实 API Key、Token、部署密钥
- 内部 handoff 和运行记录

发布前建议执行：

```bash
git status --short
git ls-files 'textbooks/*' '*.pdf' 'data/*' '.env' '.env.*'
git grep -n -E 'sk-[A-Za-z0-9]|MODELSCOPE_API_TOKEN|LLM_API_KEY=' -- README.md docs backend frontend scripts .env.example AGENTS.md
```
