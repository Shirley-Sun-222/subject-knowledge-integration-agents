# 需求分析

## 子问题分解

1. 多格式教材解析：把 PDF、Markdown、TXT、DOCX 转成统一教材和章节结构。
2. 知识点粒度定义：知识点应是教师可辨认的概念、定理、方法或现象，而不是任意关键词。
3. 重复判定标准：跨教材知识点只有在名称、定义和教学用途等价时才合并；上下位概念不直接合并。
4. 教学连贯性保障：优先保留前置依赖、章节核心概念和高频节点，低置信度删除可由教师恢复。
5. 压缩比计算：按整合后精华内容字数 / 原始教材总字数计算，目标不超过 30%。
6. RAG 分块策略：每块 500-800 字，50-100 字重叠，兼顾上下文完整性和检索精度。

## P0 验收映射

- 上传解析：前端文件列表、解析状态、章节结构。
- 知识图谱：节点和边 JSON，浏览器可交互展示。
- 跨教材整合：自动合并、保留、删除决策，显示理由和压缩比。
- RAG 问答：top-5 检索，回答带教材、章节、页码引用。
- 教师反馈：自然语言修改至少一项整合决策并更新图谱。
- 文档报告：README、需求分析、系统设计、Agent 架构说明、整合报告。
