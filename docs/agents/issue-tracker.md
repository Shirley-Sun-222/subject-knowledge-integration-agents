# Issue Tracker

The project tracks implementation work in GitHub Issues once the public repository is created.

Recommended issue shape:

- Title: concise capability or bug statement.
- Body: context, acceptance criteria, test plan, and links to relevant docs.
- Labels: use the triage mapping in `docs/agents/triage-labels.md`.

Until the GitHub remote exists, local implementation progress is recorded in `docs/实施进度.md`.
