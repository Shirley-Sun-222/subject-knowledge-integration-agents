# P2 技术报告：基于官方医学教材的 RAG 稳健性方案

## 摘要

本文围绕本项目的教材问答链路，论证一套面向官方医学教材的 **RAG 稳健性方案** 如何提升回答质量与拒答可靠性。与把所有系统亮点摊平叙述不同，本文只聚焦一个单一技术主题：在官方 7 本医学教材构成的语料库上，对比朴素向量检索、混合检索、混合检索加查询重叠保护，以及最终完整稳健性方案四个版本的行为差异。

实验显示，在当前离线实验设置下，完整稳健性方案相较纯向量检索基线，将 `pass_rate` 从 `0.360` 提升到 `0.640`，将 `source_hint_score` 从 `0.850` 提升到 `0.950`，并把 `rejection_score` 从 `0.000` 提升到 `0.800`。在调试子集 `03_生理学 + 04_医学微生物学 + 05_病理学` 上，`300` 字分块取得了最佳综合效果，因此被选为全量 7 本教材主实验的默认 chunk size。结果表明：在教材问答场景中，**混合检索决定“找得准”，查询重叠保护决定“拒得住”，而回答约束主要负责把已有检索结果组织成更可追溯的输出**。

需要强调的是，本文当前版本的实验以官方教材为语料、以自动化代理指标为评估手段，且在离线环境下运行，使用了 hash embedding fallback，并关闭了在线 LLM 生成。这使得结果更接近“保守下界”：绝对分数并不代表系统上限，但四层消融链的相对排序仍能反映方案设计的有效性。后续可在真实 embedding 与在线 LLM 配置下复核，以形成飞书终稿。

## 问题分析

### 1. 为什么教材 RAG 在这个项目里是难点

本项目不是单书检索问答，而是面向多教材知识整合的教学型系统。根据 [docs/系统设计.md](/Users/sunxueli/CodexFile/subject-knowledge-integration-agents/docs/系统设计.md:1) 与 [docs/Agent架构说明.md](/Users/sunxueli/CodexFile/subject-knowledge-integration-agents/docs/Agent架构说明.md:1)，RAG 问答承担两个目标：

1. 回答必须来自教材内部，而不是模型常识。
2. 回答必须能追溯到教材、章节和页码，才能服务教师核验。

一旦检索层出现误命中，后续回答即使语句流畅，也会把“看起来像医学知识”的无关内容包装成有引用的答案。与普通开放域问答相比，教材 RAG 更怕两类错误：

- **假阳性检索**：问的是“坏死和凋亡的区别”，却召回局部解剖学中的无关段落。
- **拒答失败**：问的是明显领域外问题，但系统仍然从医学教材中硬凑一个引用答案。

因此，本报告不把“生成看起来像答案的文本”当成核心目标，而是把三个指标视为第一性目标：

- `pass_rate`：整体答对的比例
- `source_hint_score`：引用来源是否真的落在目标主题附近
- `rejection_score`：领域外问题是否能稳定拒答

### 2. 朴素基线为什么不够

当前仓库中的生产 RAG 方案位于 [backend/app/services/rag.py](/Users/sunxueli/CodexFile/subject-knowledge-integration-agents/backend/app/services/rag.py:1)。如果把其中的保护层剥掉，只保留“向量相似度 + top-k 截断”这一最直觉的基线，会遇到三类典型问题：

- 语料中存在大量相似表述，纯向量检索容易被高频医学叙述吸走。
- 章节目录、教材前言、病例讨论等文本会带来形式上的相似，而不是语义上的相关。
- 领域外问题往往会和教材里的少量通用词发生偶然重合，导致系统错误地认为“找到了”。

在官方 7 本教材全量实验中，纯向量基线的 `rejection_score` 为 `0.000`，说明它对领域外问题完全没有防线。也就是说，**仅靠向量召回不足以支撑教材问答的稳健性要求**。

## 方案设计

### 1. 系统背景

本项目整体仍然是一个 FastAPI Orchestrator 下的模块化多 Agent 系统，但这不是本报告的主实验对象。Parser、KnowledgeExtractionAgent、AlignmentAgent、CompressionPlannerAgent、CitationQAAgent 与 ReportAgent 的职责划分见 [docs/Agent架构说明.md](/Users/sunxueli/CodexFile/subject-knowledge-integration-agents/docs/Agent架构说明.md:1)。在本文中，它们只提供上下文：

- Parser 把官方 PDF 教材解析成 `Textbook -> Chapter`
- RAG Indexer 把 Chapter 切分成 Chunk 并建立检索索引
- CitationQAAgent 在检索结果上生成带引用的回答

因此，本文评估的是 **RAG 子系统内部的设计差异**，而不是整个多 Agent 架构的优劣。

### 2.1 与公网部署相关的解析链路改造

本报告使用的是全量教材语料，因此 RAG Indexer 只应该消费 `full_ready=true` 的教材。当前生产链路已经把上传解析拆成两阶段：

- **Preview Parse**：上传后先读取 PDF TOC/书签和少量正文，默认抽取前 3 个教学章、每章最多 3 页、预览 OCR 总页数最多 9 页，用于让公网环境尽快出现可预览教材状态。
- **Full Parse**：后台继续处理整本教材，但 OCR 仍受 `OCR_MAX_PAGES` 限制，避免扫描版教材在低配机器上无上限 OCR。

这个改造不会改变本文 RAG 实验的结论，因为实验脚本仍基于全量解析后的章节集合运行；它主要解决公网演示中的交互延迟问题，并防止用户在仅有预览章节时建立误导性的 RAG 索引。换言之，**预览解析服务于上传体验，全量解析服务于 P2 报告里的 RAG 稳健性实验**。

### 2. 稳健性方案的三个组成部分

本文论证的完整稳健性方案由三层能力组成：

1. **混合检索**
   在向量相似度之外再加入 BM25 关键词分数，避免纯向量召回被表面相似语料带偏。

2. **查询重叠保护**
   通过 `_has_specific_query_overlap(...)` 对检索结果做二次约束。如果问题中的关键术语和候选 chunk 几乎没有实质重叠，即使向量分数较高，也直接拒答。

3. **回答约束**
   回答端只允许基于当前上下文作答；如果没有可靠上下文，则返回“当前知识库中未找到相关信息”。这一层不直接改善召回，但能把检索结果包装成更可追溯的输出。

### 3. 四层消融链

为了避免“终版打赢基线，但没人知道到底是哪一层起作用”，本文采用如下四层消融链：

1. 纯向量检索基线
2. 向量检索 + BM25 混合检索
3. 混合检索 + 查询重叠保护
4. 完整稳健性方案

这四个版本共享相同题集、相同教材语料、相同 `top_k=5` 设置，区别只在于检索与回答约束的开关。这样可以把增益来源尽量局部化。

### 4. 分块策略

生产代码默认在 [backend/app/services/rag.py](/Users/sunxueli/CodexFile/subject-knowledge-integration-agents/backend/app/services/rag.py:107) 使用 `700` 字左右分块，`overlap=90`。但 README 在 P2 示例中明确提到，可以通过不同 chunk size 对比来支撑 RAG 设计。因此本文额外做一组分块敏感性实验，候选为：

- `300`
- `500`
- `700`
- `900`

我们先在官方子集 `03_生理学 + 04_医学微生物学 + 05_病理学` 上比较四个分块尺寸，再把表现最好的分块尺寸带入 7 本全量教材主实验。

## 实验与结果

### 1. 数据与设置

#### 1.1 官方教材

主实验语料来自 `textbooks/` 下的 7 本官方医学教材：

- `01_局部解剖学.pdf`
- `02_组织学与胚胎学.pdf`
- `03_生理学.pdf`
- `04_医学微生物学.pdf`
- `05_病理学.pdf`
- `06_传染病学.pdf`
- `07_病理生理学.pdf`

全量实验统计如下：

- 教材数：`7`
- 解析后章节数：`3514`
- 解析后总字符数：`3,366,620`
- 在 `chunk_size=300` 下生成 chunk 数：`17,512`

调试子集只使用：

- `03_生理学.pdf`
- `04_医学微生物学.pdf`
- `05_病理学.pdf`

子集统计如下：

- 教材数：`3`
- 解析后章节数：`1994`
- 解析后总字符数：`1,745,673`
- 在 `chunk_size=700` 下生成 chunk 数：`3,787`

#### 1.2 医学 benchmark

为了让实验和官方教材域对齐，本文重建了一套医学域 25 题 benchmark，位于 `scripts/benchmark_sets/medical_official_questions.json`。题集保留现有自动评测的五类骨架：

- `fact`
- `compare`
- `reason`
- `cross_textbook`
- `out_of_domain`

每类 5 题，总计 25 题。与平均分配到 7 本教材不同，这套题集优先围绕跨教材重叠主题组织，如感染、炎症、酸碱平衡、休克、免疫防御等，从而更贴合“知识整合 + RAG 稳健性”的项目核心。

#### 1.3 运行环境说明

本轮实验的一个重要限制是：

- `using_fallback_embedding = True`
- `llm_configured = False`

也就是说，这份草稿的实验是在 **离线 hash embedding fallback + 无在线 LLM** 的保守设置下完成的。这样做有两个目的：

1. 让四层消融更多反映“检索与拒答策略”的相对差异，而不是外部模型波动。
2. 保证实验能在本地稳定复现，不依赖在线接口。

代价是：绝对指标可能低于真实 embedding + 在线 LLM 的最终表现。

### 2. 主实验：7 本全量教材上的四层消融

全量 7 本教材主实验表如下：

| 方案 | pass_rate | source_hint_score | rejection_score | avg_elapsed_ms | total_token_estimate |
| --- | ---: | ---: | ---: | ---: | ---: |
| 纯向量检索基线 | 0.360 | 0.850 | 0.000 | 352 | 16676 |
| 向量 + BM25 混合检索 | 0.480 | 0.950 | 0.000 | 960 | 18002 |
| 混合检索 + 查询重叠保护 | 0.640 | 0.950 | 0.800 | 956 | 14999 |
| 完整稳健性方案 | 0.640 | 0.950 | 0.800 | 963 | 15699 |

可以看到三个很清楚的趋势：

#### 2.1 混合检索首先解决了“找不准”

从纯向量到混合检索：

- `pass_rate`: `0.360 -> 0.480`
- `source_hint_score`: `0.850 -> 0.950`

这说明 BM25 的加入显著减少了纯向量检索被相似医学叙述误导的问题。即使在 hash embedding fallback 条件下，关键词信号依然能帮助系统把答案拉回到更贴近问题语义的教材片段。

#### 2.2 查询重叠保护决定了“拒得住”

从混合检索到“混合检索 + 查询重叠保护”：

- `pass_rate`: `0.480 -> 0.640`
- `rejection_score`: `0.000 -> 0.800`

这是整篇实验里最关键的一跳。它说明在教材问答场景中，**拒答能力不是自然出现的副产物，而必须由显式保护层强行建立**。如果没有 guard，系统几乎一定会把领域外问题误判为“领域内稀疏问题”，然后返回带引用的伪答案。

#### 2.3 回答约束更多提升可解释性，而不是主指标

从“混合检索 + guard”到“完整稳健性方案”，三项主指标基本持平：

- `pass_rate`: `0.640 -> 0.640`
- `source_hint_score`: `0.950 -> 0.950`
- `rejection_score`: `0.800 -> 0.800`

这并不意味着回答约束没有价值。恰恰相反，它说明在当前离线实验条件下，主增益主要来自检索层与拒答层，而回答约束承担的是 **输出格式化与可追溯表达** 的责任。若未来接入真实在线 LLM，这一层对答案可读性与多句综合表达的影响可能会更明显。

### 3. 分类别观察

从分类指标看，最终方案在全量语料上的表现为：

- `compare`: `1.0`
- `cross_textbook`: `0.4`
- `fact`: `0.6`
- `out_of_domain`: `0.8`
- `reason`: `0.4`

这说明当前方案最擅长两类问题：

- 有明确术语锚点的对比题
- 领域外拒答

而最薄弱的仍然是：

- 需要多步因果链的 `reason`
- 需要跨多本教材归纳整合的 `cross_textbook`

这与我们对当前离线实验设置的预期一致：检索层已经能更稳地找到局部相关内容，但更高阶的综合推理仍然受限于回答层能力和题集本身的复杂度。

### 4. 子集调参：chunk size 敏感性实验

在官方子集 `03 + 04 + 05` 上，四组 chunk size 对比结果如下：

| chunk_size | pass_rate | source_hint_score | rejection_score | avg_elapsed_ms | total_token_estimate |
| ---: | ---: | ---: | ---: | ---: | ---: |
| 300 | 0.600 | 1.000 | 0.800 | 495 | 16044 |
| 500 | 0.440 | 1.000 | 0.600 | 366 | 25048 |
| 700 | 0.480 | 1.000 | 0.600 | 323 | 32330 |
| 900 | 0.440 | 1.000 | 0.600 | 303 | 39491 |

按 `pass_rate -> source_hint_score -> rejection_score -> avg_elapsed_ms` 的规则排序，`300` 字分块被选为最终默认值，并用于 7 本全量主实验。

这个结果有两个值得注意的点：

1. **更小的 chunk 明显降低了 token 成本**
   `300` 字分块的 `total_token_estimate` 只有 `16044`，而 `900` 字分块达到 `39491`。

2. **更小的 chunk 提升了代理质量指标，但增加了检索耗时**
   `300` 字分块的 `avg_elapsed_ms` 高于 `900`，因为 chunk 数显著增多；但它带来了更高的 `pass_rate` 与 `rejection_score`。

因此，在本项目的教材场景里，chunk size 不是“越大越省事”或“越小越好”的单调问题，而是**检索精度、上下文冗余和运行代价之间的三角平衡**。当前实验表明，`300` 更适合作为稳健性优先的默认值。

### 5. 四个误例分析

#### 5.1 误例一：混合检索修正了纯向量的错误召回

题目：`坏死和凋亡有什么区别？`

- 纯向量基线：错误召回到《局部解剖学》的横结肠描述，`passed = False`
- 混合检索及其后续版本：召回到《病理生理学》第三章中关于细胞凋亡的内容，`passed = True`

这说明对于具有明确术语锚点的问题，关键词通道能有效纠正纯向量在大语料中的语义漂移。

#### 5.2 误例二：查询重叠保护显著提升了领域外拒答

题目：`今天美元兑人民币汇率是多少？`

- 纯向量和混合检索：都会从医学教材中抓到看似相关的片段并返回伪答案
- 加入 guard 后：直接返回“当前知识库中未找到相关信息”

这说明仅靠低分阈值或 top-k 截断不足以建立拒答边界；显式的术语重叠约束是必要的。

#### 5.3 误例三：混合检索改善了跨教材概念定位

题目：`酸碱平衡紊乱在生理学和病理生理学中通常涉及哪些共同概念？`

- 纯向量基线：召回到《生理学》对学科任务的泛化介绍，未能命中酸碱平衡主题
- 混合检索及后续版本：稳定命中《病理生理学》第七章关于酸碱平衡与体液 pH 的描述，`passed = True`

这说明在跨教材主题问答中，**主题关键词对召回方向的锚定** 比单纯依赖向量相似更重要。

#### 5.4 误例四：仍然存在的坏例表明关键词策略也会被污染

题目：`内环境稳态的概念是什么？`

- 纯向量基线：意外答对，命中《生理学》中“内环境、稳态和生物节律”的相关内容
- 混合检索及后续版本：却被“概念是什么”这类泛关键词拖到《病理学》中的“肿瘤的概念”，`passed = False`

这是本轮实验里最有价值的负面样本之一。它表明 BM25 不是无条件有益；当问题带有过于泛化的高频词（如“概念”）时，关键词通道反而可能把检索引向标题党式段落。未来若继续优化，应优先考虑：

- 为问题词加入更严格的医学 stopword 过滤
- 对章节标题与目录文本做更强的降权
- 在混合打分前增加 query rewriting 或术语抽取

### 6. 一个仍未完全解决的拒答漏洞

虽然 `rejection_score` 已提升到 `0.800`，但并未达到 `1.000`。最典型的残余失败是：

题目：`如何给电动汽车设计更高效的电池热管理系统？`

最终方案仍然返回了《病理学》中关于空气污染的段落。这说明当前 guard 对“医学域周边的科学/工程问题”仍不够敏感，因为它们可能和医学教材共享“系统、环境、热”等通用科学词汇。

这类失败不会被简单的术语重叠规则完全消灭，后续需要更强的域判别器或更严格的 query intent 分类。

## 局限性与未来工作

本文当前版本有四个明确局限：

1. **实验在离线 fallback embedding + 无在线 LLM 下完成**
   相对排序有参考价值，但绝对指标不是系统上限。

2. **自动评测仍是代理指标**
   `expected_source_hints` 与 `expected_answer_terms` 可以快速扩展题集，但不能完全替代人工 gold 标注。

3. **官方 7 本教材虽然已进入主实验，但 benchmark 仍偏重感染、炎症、酸碱平衡等跨教材重叠主题**
   解剖学与组织学等语料的直接问答覆盖仍相对有限。

4. **当前主实验数据来自解析链路改造前的保守快照**
   新版生产 Parser 已优先使用 PDF TOC/书签切顶层教学章，并把预览解析和全量解析拆开；这会减少目录、页眉和伪章节噪声。本文表格中的历史实验仍保留旧解析快照，适合作为保守基线，后续应在新版全量解析产物上复跑一次主实验。

下一步若继续打磨飞书终稿，我会优先做三件事：

- 用更强的医学 stopword 与 query rewriting 修正 `内环境稳态` 这类被泛词污染的问题；
