- 已将知识图谱默认展示改为章节思维导图：中心为教材或整合图谱，一级分支为章节，二级节点为知识点；同时保留原关系网络视图切换。
- 已修复图谱节点点击后的视图保持：点击节点只更新高亮和右侧详情，不再重置缩放、平移或布局位置。
- 已将知识点频次可视化改为同一蓝色由浅到深并配合节点大小变化；教材来源改由边框色、图例和详情来源列表区分。
- 已在图谱 metrics 和前端状态区暴露 LLM 抽取 / 关键词降级抽取状态，便于识别未配置 LLM API 时的知识点质量限制。
- 已新增本地 LLM 安全配置与连通性检查脚本：`scripts/configure_local_llm.py` 通过隐藏输入写入 `.env`，`scripts/check_llm.py` 检查 `complete_text` 和 `complete_json`，不打印 API Key。
- 已补充 README 中 DeepSeek、GitHub 发布和魔搭创空间环境变量部署说明。真实 API Key 只允许放入本地 `.env` 或部署平台环境变量，不能进入 Git 仓库。
- 已将 LLM 请求超时改为可配置 `LLM_TIMEOUT_SECONDS`，并限制图谱抽取输入长度和输出 token；若 LLM 超时或 schema 不兼容，图谱 metrics 会记录错误摘要并降级 fallback。
- 已将 Docker 容器默认端口改为魔搭 Docker 创空间要求的 `7860`；本地 `docker-compose.yml` 继续映射到宿主机 `8000` 便于本地访问。
- 已修复公网图谱构建导致的 SQLite 长事务锁：LLM 章节抽取移到数据库写事务外，写入阶段只保留短事务；同时增加 SQLite busy timeout，降低并发写入时的 `database is locked` 风险。
- 已让前端公网图谱构建默认请求 `max_chapters=3`，后端仍受 `GRAPH_MAX_CHAPTERS` 总上限保护，返回 metrics 会标明是否截断；完整图谱仍可通过脚本或后续后台任务方案扩展。
- 已将公网长写接口改为后台任务模式：上传解析、单本图谱、跨教材整合、RAG 建索引和 PDF 报告生成统一返回 `202 Accepted`，前端通过 `/api/tasks` 轮询状态。
- 已新增运行时 adapter：把任务状态、教材/图谱/整合/RAG 的 SQLite 状态访问，以及上传教材与报告产物路径，分别收拢到 `runtime/store.py` 和 `runtime/files.py`。
- 已将前端任务轮询与恢复逻辑从 `main.tsx` 中抽离到独立 workflow module，页面刷新后可恢复 `queued/running` 任务。
- 已新增 ADR `docs/adr/0002-single-container-background-tasks.md`，记录为什么选择单容器后台任务，而不是继续同步长请求、引入外部队列或转向 GitHub Pages。
- 已继续优化公网性能：`/api/textbooks` 改为教材摘要接口，不再返回完整章节正文；图谱抽取支持按章节并发执行；`GraphCanvas` 与 Cytoscape 改为懒加载，首屏主 bundle 明显缩小。
- 已继续优化上传与图谱链路：PDF 解析任务增加页级进度和 `digital / mixed / scanned` 快速判定；相同教材与相同章节上限的单本图谱请求会命中缓存并直接复用已有结果。
- 已进一步轻量化 Docker 默认镜像：移除未使用的 `faiss-cpu`、`rank-bm25`、`requests`，默认不预装 Playwright Chromium，默认 `PDF_RENDERER=reportlab`，需要高保真浏览器 PDF 时再通过构建参数显式开启。
- 已继续优化公网压力：任务轮询改为前快后慢的自适应策略；RAG 建索引支持完整 cache hit 和按章节增量复用，避免每次都全量重建。
- 已把公网默认数据模型从全站共享切到会话工作区：每个浏览器访问者获得独立教材状态，支持教材选择性删除，并通过 TTL 自动清理。
- 已新增会话级 LLM 配置：访问者可临时填写自己的 Base URL / API Key / Model，优先级高于部署者全局模型配置。
- 已把图谱范围交互改为“预览图谱 / 全量图谱”显式切换，并在标准版前端隐藏低质量 PDF 下载入口。
