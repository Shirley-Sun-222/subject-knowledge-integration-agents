# 学科知识整合智能体领域上下文

## 领域目标

系统帮助教师把多本同学科教材整合成体量不超过原始 30% 的精华知识版本，同时保留可追溯引用、知识图谱结构和教师可修订的整合决策。

## 核心术语

- **教材 Textbook**：用户上传的 PDF、Markdown、TXT 或 DOCX 文件，是系统处理的原始知识来源。
- **章节 Chapter**：从教材中识别出的结构单元。若自动识别失败，系统创建虚拟章节以保证处理链路不断。
- **文本块 Chunk**：RAG 检索的最小证据单元，约 500-800 字，带教材、章节、页码元数据。
- **知识点 KnowledgeNode**：从章节抽取出的概念、定理、方法或现象，必须包含名称、定义、分类、章节和出处。
- **知识关系 KnowledgeEdge**：知识点之间的教学关系，支持前置依赖、并列、包含、应用。
- **整合决策 IntegrationDecision**：跨教材去重提纯的可解释动作，包括合并、保留、删除。
- **精华内容 Integrated Content**：整合后保留给教师和学生使用的摘要化知识内容，用于计算 30% 压缩比。
- **引用 Citation**：RAG 回答中的来源证据，只能来自 chunk 元数据，不能由模型编造。
- **教师反馈 TeacherFeedback**：教师对整合决策的自然语言修订请求，例如保留、删除、拆分或合并。
- **OCR fallback**：当 PDF 页面没有可抽取文本时，Parser 调用 Tesseract 对页面渲染图像进行识别，输出仍进入同一 Chapter schema。

## 不变量

- RAG 答案必须只基于检索到的 chunk。
- 页码、章节、教材名称必须从解析和 chunk 元数据中读取。
- 压缩比按整合后精华内容字数除以原始教材总字数计算，不按节点数计算。
- Agent 之间通过数据库、文件和 schema 共享状态，不通过聊天历史共享状态。
- 上传教材、解析结果、索引和报告属于后端服务端状态；浏览器只负责上传文件和展示结果。
- 扫描版 PDF 不能让后续链路静默失败：OCR 失败时教材状态必须记录失败原因，图谱构建不能依赖空章节继续伪造结果。
- 图谱构建可以按章节数截断以保护云端运行时间，但接口必须返回已处理章节数、总章节数和截断标志。
